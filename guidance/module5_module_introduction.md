# Module 5 Introduction: Basic Data Transformation with Pandas

After profiling, cleaning, and improving your workflow habits, this module moves into basic data transformation. Cleaning asks what is wrong with the data. Transformation asks how the data should be shaped for the analysis we want to do.

This week focuses on the basic grammar of Pandas transformation: filtering rows, selecting and reordering columns, renaming variables, sorting observations, creating derived columns, and exporting a transformed dataset. These are everyday analyst moves, and they prepare you for more complex work later.

## This Week You Will

- filter rows using rules and conditions;
- sort records to inspect important or extreme cases;
- select, reorder, and rename columns;
- create derived variables from existing variables;
- export a transformed CSV file;
- explain how a transformation changes the meaning or usability of a dataset.

## Lab Focus

The guided lab uses the `nycflights13` flights dataset. Each row represents one scheduled flight from a New York City airport in 2013. You will practice focused transformations without adding aggregation, reshaping, joins, or formal documentation yet.

## Important Boundary

This module stays intentionally focused. We are not covering heavy string cleaning, categorical recoding, aggregation, reshaping, SQL joins, or full validation this week. Those topics appear in later modules.

## Project Connection

Project Milestone 1 is due around this module. That milestone focuses on your initial data profile and basic cleaning plan. The transformation lab is separate, but it should help you think more clearly about how raw data becomes useful analytical data.
